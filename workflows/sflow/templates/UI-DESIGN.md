## 1. Visual Direction

### Tone

**Minimal** вЂ” Clean, purposeful, and restrained. Every element earns its place.

### Reference Products

- Apple Human Interface вЂ” spatial clarity and typographic hierarchy
- Linear вЂ” keyboard-first workflows with zero visual noise
- Vercel Dashboard вЂ” information density without clutter

### Core Emotions

- **Confidence**: Users feel the product is reliable and well-built
- **Clarity**: Information is immediately scannable, no ambiguity
- **Calm**: The interface never shouts; it guides

## 4. Interaction Patterns

### Hover State

- Background lightens by oklch lightness +0.03
- Transition: 150ms ease-out
- Cursor: pointer for interactive elements

### Focus State

- 2px solid ring using `--color-primary`
- Offset: 2px from element boundary
- Visible on keyboard navigation only (use `:focus-visible`)

### Active State

- Background darkens by oklch lightness -0.02
- Scale: 0.98 for buttons (subtle press effect)
- Transition: 75ms ease-in

### Disabled State

- Opacity: 0.4
- Cursor: not-allowed
- No hover/focus/active transitions
- ARIA: `aria-disabled="true"`

### Loading State

- Spinner replaces content (not overlay)
- Skeleton screens for initial page loads
- Disabled interactions during loading
- ARIA: `aria-busy="true"`, `aria-live="polite"`

| `xs` | 360px | Single column, stacked navigation |
| `sm` | 768px | Two-column where needed, hamburger menu |
| `md` | 1024px | Sidebar visible, multi-column grids |
| `lg` | 1440px | Full layout, max-width container |

### Responsive Rules

- Content never exceeds 1440px вЂ” center with auto margins beyond
- Typography scales down one step below 768px
- Navigation collapses to hamburger below 768px
- Tables switch to card-list below 768px
- Forms remain single-column at all widths

## 6. Accessibility Guidelines

### WCAG 2.1 AA Checklist

- [ ] **Color Contrast**: Text contrast ratio >= 4.5:1 (normal text), >= 3:1 (large text)
- [ ] **Non-Text Contrast**: UI components and graphical objects >= 3:1 against adjacent colors
- [ ] **Keyboard Navigation**: All interactive elements reachable via Tab, operable via keyboard
- [ ] **Focus Indicators**: Visible focus ring on all interactive elements (use `:focus-visible`)
- [ ] **Screen Reader Labels**: Every form input has associated `<label>` or `aria-label`
- [ ] **Heading Hierarchy**: Headings follow strict h1 > h2 > h3 order, no skipped levels
- [ ] **Image Alt Text**: All meaningful images have descriptive alt text; decorative images use `alt=""`
- [ ] **ARIA Landmarks**: Use `<main>`, `<nav>`, `<aside>`, `<footer>` for page structure
- [ ] **Error Identification**: Form errors are associated with inputs via `aria-describedby`
- [ ] **Reduced Motion**: Respect `prefers-reduced-motion` вЂ” disable animations when active
- [ ] **Touch Targets**: Minimum 44x44px for all interactive elements on touch devices
- [ ] **Language Declaration**: `<html lang="...">` set correctly for the content language

## 7. Placeholder Strategy

### Rules

1. **No emoji** in placeholder content вЂ” use text labels only
2. **No fabricated data** вЂ” use clearly marked placeholders like "Item Name" or "Sample Title"
3. **No fake avatars** вЂ” use initials-based placeholders or a generic user icon
4. **Honest empty states** вЂ” use `v-if` guards; show meaningful empty messages, not dummy content
5. **No Lorem Ipsum** вЂ” write purposeful placeholder text that describes what should go there
6. **No decorative images** вЂ” if an image is needed, use a labeled placeholder rectangle

### Placeholder Format

```
[Placeholder: <description of expected content>]
```

Example: `[Placeholder: User profile photo]` instead of a random stock image.

## 9. Do's and Don'ts

### Do

- Use OKLCH color format for all color definitions вЂ” perceptually uniform and accessible
- Define spacing as multiples of the base unit (4px) вЂ” never use arbitrary pixel values
- Use semantic color tokens (`--color-success`, `--color-error`) instead of raw hex values
- Provide real, meaningful content in designs вЂ” even in early prototypes
- Test all interactive states: hover, focus, active, disabled, loading
- Use `v-if` guards for empty states to prevent flash of empty content
- Maintain consistent heading hierarchy throughout the page
- Use system font stacks that gracefully degrade across platforms

### Don't

- Never use `border-left` as a decorative stripe вЂ” use background color or a dedicated accent element
- Never start tags/labels with `#` вЂ” use plain text or a styled chip component
- Never use Inter, Roboto, or Arial as the primary typeface вЂ” these signal AI-generated design
- Never use pure black (#000000) for text вЂ” use the foreground token with slight warmth
- Never use pure white (#ffffff) as a background вЂ” use the background token with slight warmth
- Never add animations without `prefers-reduced-motion` fallback
- Never use Lorem Ipsum or random placeholder text вЂ” write purposeful descriptions
- Never leave empty states unguarded вЂ” always wrap with `v-if` + data check
- Never use emoji as UI elements or in placeholder content
